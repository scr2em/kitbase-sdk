Kitbase Flags · QA Console

Pick an evaluation mode to test. Both pages load @kitbase/flags from unpkg.

Remote

Remote evaluation

The default mode. Each evaluation hits the API; results are cached client-side per the configured TTL.

  • Cache TTL + persistent localStorage
  • Identity (identify, resetIdentity, anonymousId)
  • Boolean / string / number / json evaluation
  • Snapshots & OpenFeature provider
 Local

Local evaluation

Configuration is fetched once and refreshed periodically. Flag evaluations resolve in-memory with no network round-trip.

  • Refresh interval, initial config bootstrap
  • Lifecycle: initialize(), waitUntilReady(), isReady(), close()
  • Configuration introspection (getConfiguration, getFlagKeys, hasFlag, refresh)
  • Lifecycle events (on) and flag-change subscriptions

Need a different version? Replace https://unpkg.com/@kitbase/flags in the page <script> tag with a pinned URL like https://unpkg.com/@kitbase/flags@0.2.0.
